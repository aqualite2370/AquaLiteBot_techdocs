# 文档更新流程

当前线上站点读取的是：

- `frontend/public/data/documents.json`

源数据维护在：

- `backend/data/documents.json`

现在只保留一份自动同步和发布脚本：

- `一键更新并发布文档.bat`

## 推荐用法

直接双击 `一键更新并发布文档.bat`。

它会自动执行：

1. 从有道分享链接导入最新文档
2. 规范化文档内容
3. 应用手工精修规则
4. 同步数据到前端静态文件
5. 暂存文档生成结果
6. 自动提交
7. 自动推送到 GitHub 当前分支

推送到 GitHub 后，Cloudflare Pages 会按仓库配置自动重新部署。

## 注意

- 如果没有任何文件变化，脚本会提示没有需要提交的内容并退出。
- 如果 `backend\venv\Scripts\python.exe` 不存在，脚本会自动尝试使用系统 `python`。
- 脚本只会暂存 `backend/data/documents.json` 和 `frontend/public/data/documents.json`，不会把其它未提交改动一起推上去。
- 如果 `git push` 失败，本地提交仍然会保留，可以处理网络或权限问题后手动运行 `git push`。
