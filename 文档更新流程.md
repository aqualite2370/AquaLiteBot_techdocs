# 文档更新流程

当前 Cloudflare Pages 线上站点读取的是：

- `frontend/public/data/documents.json`

而你现有的导入脚本原本更新的是：

- `backend/data/documents.json`

现在已经帮你补好自动同步，所以以后你可以继续按现有脚本习惯更新。

## 最推荐的更新方式

直接双击：

- `更新文档并同步前端.bat`

它会按这个顺序执行：

1. 从有道分享链接重新导入最新文档
2. 规范化文档内容
3. 应用你的手工精修规则
4. 把最终结果同步到前端静态数据

## 更省事的一键发布方式

如果你希望更新完就直接进入 Git 提交流程，直接双击：

- `一键更新并发布文档.bat`

它会自动执行：

1. 更新文档
2. 同步前端数据
3. `git add .`
4. 让你确认提交信息
5. 询问你是否立即 `git push`

## 如果你只想快速同步原始文档

直接双击：

- `sync_note.bat`

现在它除了更新 `backend/data/documents.json`，还会自动同步到：

- `frontend/public/data/documents.json`

## 更新后如何发到线上

文档更新完成后执行：

```powershell
cd "C:\Users\1\Desktop\前后端相关\vue_fastapi_技术文档"
git add .
git commit -m "Update docs"
git push
```

推送到 GitHub 后，Cloudflare 会自动重新部署。

## 你以后真正需要记的只有这两种

### 方案 1：完整更新，推荐

- 双击 `更新文档并同步前端.bat`

### 方案 2：一键更新并发布

- 双击 `一键更新并发布文档.bat`

### 方案 3：更新完再手动发布到线上

```powershell
git add .
git commit -m "Update docs"
git push
```
